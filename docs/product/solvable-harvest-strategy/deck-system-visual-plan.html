Soli architecture artifact - deck system v2

Generate decks once, identify them exactly, and select solvable deals by draw mode.

This plan separates random generation from durable deal identity. Generators can change. Exact permutation IDs keep history, replay, sharing, and solver validation stable.

Recommendation: use an app-owned generator for random-looking decks, encode every full 52-card permutation as an exact ID, solve batches offline with Lonelybot, and store history against the exact ID plus draw-specific solvability.

Runtime Flow

Runtime generation stays simple. The app does not need the solver for v2 deals.

Random Game

  • Generate a `soli-v1` seed.
  • Create deck and exact ID.
  • Deal normally.
  • History stores exact ID, but solvable flag is false or unknown.

Solvable Game

  • Read current draw count.
  • Pick exact ID from offline index with matching draw mask.
  • Decode exact deck and deal full tableau plus stock.
  • History stores exact ID and `solvableDraws`.
Runtime app code should not call Lonelybot for normal new games. The solver is useful offline first, then later for hints.

Decision Board

Defaults to keep v2 tight and avoid unnecessary migration complexity.

Do Now

  • Use full deals for all draw counts.
  • Store exact IDs in history.
  • Ship a compact solvable index.
  • Keep old history readable.

Decide Once

  • PCG vs xoshiro for `soli-v1`.
  • Exact ID string encoding.
  • Minimum solved index size.
  • Legacy upgrade stock policy.

Avoid

  • `Math.random` for durable games.
  • Library-default RNG as identity.
  • Tableau-only v2 deals.
  • Claiming draw solvability without proof.

Implementation Phases

Split the work so the dataset migration does not get tangled with in-app hints.

1 Identity

  • Implement exact encode/decode in TypeScript.
  • Fixture against Lonelybot.
  • Pick exact ID string format.

2 Harvest

  • Generate large seed batches.
  • Run Lonelybot Draw 1-5.
  • Build compact index.

3 App

  • Create games from exact IDs.
  • Select by draw mask.
  • Update history labels.

4 Later

  • Rust native solver spike.
  • Current-position hints.
  • Undo to solvable position.
