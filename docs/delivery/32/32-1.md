## Requirements
- On iOS, place the drawer control in the header’s leading slot on the Play screen.
- Keep the `Soli` title centered using the standard iOS header layout.
- Keep `New Game` directly visible on iOS, but style it as a compact, unobtrusive navigation-bar action rather than a filled pill.
- Ensure iOS header controls respect 44pt minimum tap targets and compact navigation-bar glyph/text sizing.
- Keep the current Android Play header layout and behavior unchanged.

## Implementation Plan
1. Add iOS-only header option branching in `app/(tabs)/index.tsx`.
2. Extract constants for navigation control hit targets, icon glyph sizing, and text sizing so the header math stays explicit.
3. Render the iOS drawer action in `headerLeft` and the visible `New Game` action in `headerRight`.
4. Leave the Android header cluster as-is and preserve the current new-game confirmation flow.

## Test Plan
- Run `yarn typecheck:tsc` to confirm the iOS-specific header option branches compile cleanly.
- Run `yarn lint app/'(tabs)'/index.tsx app/'(tabs)'/_layout.tsx` if needed via the project lint command to catch header-option issues.
- Manually inspect the Play screen on iOS-sized layouts to confirm the title stays centered and `New Game` remains visible without overlap.
- Manually confirm Android still shows the existing trailing header cluster.
