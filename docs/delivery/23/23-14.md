## Requirements
- Remove the `borderColor` override capability from `CardView`/`CardVisual` and all call sites, relying on the standard card border defined in styles.
- Restore the original fixed typography and positioning for card ranks, suits, and symbols.
- Keep a reusable card visual helper for future static usage without reintroducing dynamic styling.
- Ensure no consumer relies on the removed border prop and adjust tests/type checks accordingly.

## Implementation Plan
1. Update documentation and exports to reflect the new helper while removing borderColor props.
2. Simplify `CardView` and related components to use static styles and constants.
3. Adjust any callers or hooks that previously supplied border colors.
4. Run TypeScript checks to confirm clean integration.
