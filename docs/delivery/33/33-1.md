## Requirements
- Use the connected Android source fonts rather than substituting a visually similar family.
- Generate a static Android-derived rank font at weight 700 so React Native does not need to synthesize or resolve card rank weight dynamically.
- Preserve the approved large suit-symbol footprint and exact fat-heart shape even if a plain `Text` font render proves too small or too shallow on one or both platforms.
- Keep the app-wide typography unchanged outside the card rank/suit text nodes.
- Document the package usage for `fonttools` and `expo-font` with fresh, dated guide files before relying on them in implementation.

## Implementation Plan
1. Research the official `fonttools`, `expo-font`, and React Native font docs to confirm the supported implementation path.
2. Create a reproducible script that generates a static bold Roboto rank font from the Android variable source font and capture the four suit symbols from the connected Android device's `NotoColorEmoji.ttf`.
3. Commit the generated card assets under `assets/fonts/` and `assets/card-suits/`.
4. Load the rank font during app startup and expose the card-only font-family constant.
5. Apply the explicit rank font plus Android-derived suit assets in `CardView` while leaving card sizes, spacing, and animation behavior unchanged.

## Test Plan
- Confirm the generated rank font resolves `A234567890JQK` and no suit glyphs.
- Confirm the generated suit assets resolve to the intended Android glyph shapes.
- Run project typecheck and lint commands after wiring the fonts into the app.
- Launch the app on Android and iOS to confirm the board still renders and only card glyphs changed.

## Verification
- Android source-font verification:
  - `adb -s emulator-5554 shell sed -n '1,220p' /system/etc/fonts.xml | rg -n 'family name=\"sans-serif\"|font weight=|NotoColorEmoji|Roboto'` confirms `sans-serif` weight `700` resolves through `Roboto-Regular.ttf` and the Android font set includes `NotoColorEmoji.ttf`.
  - Both connected Android targets expose `Roboto-Regular.ttf`, `RobotoStatic-Regular.ttf`, and `NotoColorEmoji.ttf` under `/system/fonts`.
- Asset generation:
  - Installed `fonttools` in a temporary venv and ran `/tmp/soli-fonttools-venv/bin/python scripts/build-card-fonts.py`.
  - Generated `assets/fonts/CardRankRoboto700.ttf`.
  - Exported trimmed transparent suit assets from Android `NotoColorEmoji.ttf` into `assets/card-suits/`.
- Glyph-shape verification:
  - `hb-shape assets/fonts/CardRankRoboto700.ttf 'A234567890JQK♣♦♥♠'` resolves the rank glyphs and returns missing glyphs for the four suits.
  - Render comparisons across `Inter-Regular.ttf`, `NotoSansSymbols-Regular-Subsetted2.ttf`, and `NotoColorEmoji.ttf` showed the approved fat-heart shape only in `NotoColorEmoji.ttf`.
  - The checked-in suit assets were exported from that Android emoji source and trimmed so the center and corner suits keep the approved larger footprint on both platforms.
- App validation:
  - `yarn typecheck` passed.
  - `yarn typecheck:fallback` passed.
  - `yarn lint` passed.
