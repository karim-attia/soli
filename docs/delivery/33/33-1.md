## Requirements
- Use the connected Android source fonts rather than substituting a visually similar family.
- Preserve the original Android text-based card renderer instead of switching cards to custom images or SVGs.
- Keep the shipped rank font as an Android-derived static Roboto Bold generated from the connected device's `Roboto-Regular.ttf` source and keep the shipped suit glyphs on exact Android `NotoColorEmoji` exports so the wide heart stays identical cross-platform.
- Keep the app-wide typography unchanged outside the card rank/suit text nodes.
- Document the source-backed card-font workflow and native font-loading approach used by the app.

## Implementation Plan
1. Research the official React Native and Expo font docs to confirm the supported implementation path for bundled native fonts.
2. Pull the Android source font and suit glyph sources from the connected device and generate/export the app assets.
3. Register the rank font natively through the Expo font plugin and expose the card-only font-family constant.
4. Apply the explicit rank font and exact Android suit glyph assets in `CardView` while leaving card sizes, spacing, and animation behavior unchanged.
5. Clean-rebuild both targets and compare the card top-row and center-symbol rendering against the approved Android look.

## Test Plan
- Confirm the bundled rank font is generated from the connected device's Android Roboto source and the suit glyph assets match the approved Android emoji-source symbols.
- Run project typecheck and lint commands after wiring the fonts into the app.
- Launch the app on Android and iOS to confirm the board still renders and only card glyphs changed.
