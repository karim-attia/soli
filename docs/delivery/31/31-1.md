## Requirements

- Add `oxlint`, `oxfmt`, and `@typescript/native-preview` to project tooling.
- Keep package scripts and config aligned with current TypeScript 7, Oxc, and Expo guidance.
- Use `tsgo` as the sole root project type checker without a duplicate TypeScript 6 fallback.
- Remove Biome configuration from the repository.
- Create package guide docs for `tsgo`, `oxlint`, and `oxfmt` with date-stamped sources and usage snippets.

## Implementation Plan

1. Inspect `animals` project to capture the exact script/config/dependency pattern.
2. Research official documentation for each package and capture practical usage constraints.
3. Update `soli` package scripts and dev dependencies; configure `.oxlintrc.json`, `.oxfmtrc.json`, and one Expo-based `tsconfig.json`.
4. Remove `biome.json` and verify no Biome usage remains in project configuration.
5. Run format, lint, native typecheck, tests, Expo, Tamagui, and web export verification.
