 ## Requirements
 - Stop dispatching periodic timer updates that mutate full `GameState` every second.
 - Keep elapsed time visibly updating in the statistics HUD while timer is running.
 - Ensure timer correctness for start, pause/background, auto-up, and win flows.
 - Preserve existing game behavior and animation semantics outside timer update frequency.
 - Re-run `scripts/perf-baseline.sh` multiple times and record before/after metrics.
 
 ## Implementation Plan
 1. Remove reducer tick interval dispatch from `useKlondikeTimer` while preserving start/stop logic.
 2. Move live timer display ticking into the HUD component (localized rerender scope).
 3. Update statistics model wiring from `useKlondikeGame` to support HUD-local ticking.
 4. Verify typecheck and smoke behavior.
 5. Run baseline script for repeated runs and compare to previous baseline.
 
 ## Test Plan
 - Manual: start a game and confirm timer updates while running, pauses in background, and stops on auto-up/win.
 - Manual: interact with board while timer runs and confirm no gameplay regressions.
 - Scripted: run `ADB_SERIAL=<serial> ./scripts/perf-baseline.sh 3` and capture summary table.
 - Build: run `yarn typecheck`.
